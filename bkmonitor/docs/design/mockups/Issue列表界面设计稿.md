| 元素 | 说明 |

**样式说明：**

- 数字可点击，点击后弹出抽屉或弹窗展示该类型下的具体资源列表

```
集群: 100
主机: 124
```

或

```
集群: 100
namespace: 30
pod: 30
```

**样式规范：**
- 资源类型名称和数字之间用冒号和空格分隔
- 不同资源类型之间换行展示
- 单元格内左对齐，垂直居中

## 4. 交互说明

### 4.1 筛选交互
- 左侧快捷筛选项点击后立即触发列表刷新
- 支持多选组合筛选
- 选中项高亮显示

### 4.2 排序交互
- 表头点击切换排序（升序/降序）
- 当前排序列显示 ▼ 或 ▲ 图标

### 4.3 批量操作
- 勾选行复选框后，批量操作下拉可用
- 支持全选/反选

### 4.4 行操作
- 点击"标为已解决"按钮，将 Issue 状态变更为已解决
- 行悬停显示操作按钮

### 4.5 趋势图
- Sparkline 柱状图展示告警时间分布
- 悬停显示具体数值
- 点击可进入详情页查看完整趋势

### 4.6 影响范围交互
- 显示各资源类型的统计数量（如集群: 100）
- **数量数字为蓝色，表示可点击**
- 点击数字后展开抽屉，显示该资源类型下的具体资源列表
- 支持展示的资源类型包括：集群、主机、namespace、pod、node 等

### 4.6 指派负责人弹窗

```
┌─────────────────────────────────────┐
│  指派负责人                      [X] │
├─────────────────────────────────────┤
│                                     │
│  负责人 *                           │
│  ┌───────────────────────────────┐  │
│  │ 请输入                         │  │
│  └───────────────────────────────┘  │
│                                     │
│                                     │
│           [确定]    [取消]          │
│                                     │
└─────────────────────────────────────┘
```

**交互说明：**
- 点击"未指派"或负责人区域弹出指派弹窗
- 支持输入搜索选择负责人
- 必填项，确定后更新 Issue 负责人

## 5. 字段详细说明

根据产品标注，各字段详细说明如下：

| 标注 | 字段名称 | 说明 |
|------|----------|------|
| a | **Issues 名称** | 蓝色字，默认是告警策略名称，可点击，点击出现详情抽屉 |
| b | **关键堆栈信息** | 这里用英文占位，如 `NullpointerException` |
| c | **Issues 类型** | 新问题、回归问题 |
| d | **告警事件数量** | Tag 显示，如 🔴 123 |
| e | **标签** | 告警策略的标签，如"集群告警"、"测试环境"、"BCS" |
| f | **最后出现时间** | 最后的告警产生时间，显示绝对时间+相对时间，支持排序 |
| g | **最早发生时间** | 最早的告警产生时间，显示绝对时间+相对时间，支持排序 |
| h | **趋势** | 按告警时间产生的趋势统计，Sparkline 柱状图 |
| i | **影响范围** | 显示各类受影响的资源统计数量，格式为"资源类型: 数量"，**数量数字使用蓝色显示**。支持展示的资源类型包括：集群、主机、namespace、pod、node 等。点击数字可展开查看具体的资源列表详情 |
| j | **优先级** | 支持手动调整优先级，高(红色)/中(橙色)/低 |
| k | **状态** | 支持手动调整状态：待审核(初始状态，负责人为空显示"未指派")、未解决(设置负责人后自动变更为待解决)、已解决(手动标记已解决后显示) |
| l | **负责人** | 支持修改，支持表头筛选 |
| m | **标记已解决** | 操作按钮，点击将 Issue 状态变更为已解决 |

## 6. 优先级与状态交互

### 6.1 优先级选择下拉

```
┌───────────┐
│  [高] 高  │  ← 红色
│  [中] 中  │  ← 橙色
│  [低] 低  │  ← 蓝色
└───────────┘
```

- 点击优先级徽章弹出下拉选择
- 选择后立即更新 Issue 优先级

### 6.2 状态流转

```
┌──────────┐      设置负责人       ┌──────────┐
│  待审核   │ ───────────────────→ │  未解决   │
│ (初始状态)│                      │(待解决)  │
└──────────┘                      └────┬─────┘
      │                                │
      │         标记已解决             │
      └────────────────────────────────┘
                                     ↓
                               ┌──────────┐
                               │  已解决   │
                               └────┬─────┘
                                    │
                                    │ 归档
                                    ↓
                               ┌──────────┐
                               │   归档    │
                               │ (只读状态)│
                               └──────────┘
```

**状态说明：**
- **待审核**：初始状态，此时【负责人】为空，显示"未指派"
- **未解决**：设置【负责人】之后，状态自动变更为"待解决"（显示为"未解决"）
- **已解决**：手动标记已解决后，状态显示为"已解决"
- **归档**：已解决的 Issue 可进一步归档，归档后 Issue 进入只读状态，不再活跃显示
