# 产品概述

> 返回 [目录](./README.md)

在 bkmonitor/alarm_backends/ 模块中实现基于配置的数据流处理框架，封装告警处理全流程，支持通过配置灵活编排处理节点，提供数据库配置管理和脚本管理接口。

## 核心功能

| 功能 | 说明 |
|------|------|
| **事件丰富** | 支持多维数据增强和元数据补充 |
| **过滤** | 基于规则条件的事件过滤机制 |
| **抑制** | 事件抑制和屏蔽逻辑 |
| **限流** | QoS 限流和流量控制 |
| **熔断** | 熔断机制和故障隔离 |
| **收敛** | 事件收敛和去重聚合 |
| **降噪** | 智能降噪和噪音过滤 |
| **屏蔽** | 告警屏蔽和故障期管理 |
| **处理** | 告警处理和状态转换 |
| **通知** | 多渠道通知和消息推送 |
| **优先级检查** | 告警优先级评估和调整 |
| **级别检查** | 告警级别验证和校验 |
| **动作触发规则** | 自定义动作触发条件和规则配置 |
| **配置驱动处理** | 支持 JSON/YAML 格式的流程配置 |
| **配置版本管理** | 支持流程模板跨项目共享和版本控制 |

## 技术栈选择

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **编程语言** | Python 3.10+ | 现有系统技术栈 |
| **Web 框架** | Django + Django REST Framework | 现有系统技术栈 |
| **数据存储** | Redis / ElasticSearch / PostgreSQL | 缓存 / 持久化 / 配置管理 |
| **消息队列** | Kafka | 事件流 |
| **配置格式** | JSON / YAML | 便于人工编辑和机器解析 |
| **分布式支持** | Celery | 异步任务 |
| **分布式通信** | HTTP / gRPC / Kafka | 多协议支持，可根据场景选择 |
