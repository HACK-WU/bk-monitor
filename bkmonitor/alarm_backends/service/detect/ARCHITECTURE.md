# Detect 模块架构文档

## 1. 模块概述

Detect 模块是蓝鲸监控平台告警后端的核心检测模块，负责对标准化的时序数据进行异常检测判断，输出异常事件供下游 Trigger 模块消费。

### 1.1 核心职责

- 接收 Access 模块产生的待检测数据
- 执行多种检测算法进行异常判断
- 支持多级别告警和多算法组合检测
- 输出异常数据供告警触发

## 4. 核心组件详解

### 4.1 处理层组件

| 组件 | 文件 | 职责 |
|------|------|------|
| DetectHandler | handler.py | 同步检测处理入口，监听 Redis 信号队列 |
| DetectCeleryHandler | handler.py | 异步批量检测处理，支持 Celery 任务分发 |
| DetectProcess | process.py | 检测流程控制器，协调数据拉取、检测、推送 |
| run_detect | tasks.py | Celery 普通检测任务 |
| run_detect_with_sdk | tasks.py | Celery SDK智能检测任务 |

### 4.2 核心数据模型

| 组件 | 文件 | 职责 |
|------|------|------|
| DataPoint | core.py | 数据点封装，包含 value、timestamp、dimensions 等 |
| AnomalyDataPoint | core.py | 异常数据点，包含异常消息、检测器信息等 |

### 4.3 算法基类

| 组件 | 文件 | 职责 |
|------|------|------|
| Algorithms | strategy/__init__.py | 检测算法抽象基类 |
| ExprDetectAlgorithms | strategy/__init__.py | 表达式检测算法 |
| BasicAlgorithmsCollection | strategy/__init__.py | 算法集合基类 |
| RangeRatioAlgorithmsCollection | strategy/__init__.py | 同比环比算法基类 |
| HistoryPointFetcher | strategy/__init__.py | 历史数据获取器 |
| SDKPreDetectMixin | strategy/__init__.py | SDK智能预检测混入类 |

### 4.4 具体检测算法

| 算法 | 文件 | 说明 |

## 9. 扩展开发指南

### 9.1 新增检测算法

1. 在 `strategy/` 目录下创建新的算法文件
2. 继承 `BasicAlgorithmsCollection` 或 `RangeRatioAlgorithmsCollection`
3. 实现 `gen_expr()` 方法生成检测表达式
4. 实现 `extra_context()` 方法提供额外上下文变量
5. 如需历史数据，实现 `get_history_offsets()` 方法

```python
# 示例：自定义阈值算法
from alarm_backends.service.detect.strategy import BasicAlgorithmsCollection, ExprDetectAlgorithms

class CustomThreshold(BasicAlgorithmsCollection):
    desc_tpl = "自定义阈值检测: 当前值 {value}, 阈值 {threshold}"
    
    def gen_expr(self):
        threshold = self.validated_config.get("threshold", 0)
        yield ExprDetectAlgorithms(
            f"value > {threshold}",
            self.desc_tpl
        )
```

### 9.2 新增二次确认策略

1. 在 `double_check_strategies/` 目录下创建策略文件
2. 继承 `DoubleCheckStrategy` 基类
3. 实现 `double_check()` 方法
4. 在 `__init__.py` 中注册策略
